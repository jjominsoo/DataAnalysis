데이터 변화
데이터 양,소스 증가함으로써 보관(data lake), 처리(빅데이터 처리)기술들이 필요해지고 해당 직군들도 발전했다 >> 새로운 형태의 데이터 관리가 필요함 == 데이터 거버넌스 (데이터 수집부터 분석, 저장까지 하는 전체적인 프로세스)
또한 이로 인해 발생될 수 있는 문제들도 해결해야한다. .. 개인정보, 다양한 해석, 비슷한 대시보드/테이블(정보 과잉으로 인한 비용증가), 프로세스(ELT ETL) 오너십, 메타 데이터(데이터 태그) 부족, Data Silo(부서별 독립적 데이터)
1. 무분별한 개인정보 전파
해결 : 개인정보 생성과 동시에 태깅, 접근권한 제어 및 로깅, 추출과 삭제 자동화, GDPR(General Data Protection Regulation)
2. 다양한 해석
해결 : KPI 정의사전(데이터 용어집), 애초에 같은 데이터 기반으로 사용
3. 너무 많은 대시보드와 비슷한 테이블들
해결 : 데이터 카탈로그, 검색엔진 도입 = data discovery >> 메타 데이터, 데이터셋 오너 관리 + 주기적인 청소작업
4. 불분명한 데이터 오너십 == 메타 데이터 
> 품질 이슈로 이어짐
해결 : 메타 데이터 관리
5. 메타 데이터(데이터에 대한 데이터) 부족
해결 : 중요 데이터 별로 오너 지정, 데이터 리니지(데이터 흐름의 시각화) 파악, 데이터 별 분류(PII)
6. Data Silo
동일한 데이터를 여러팀에서 반복
해결 : 조직 전반에 메타 데이터 관리/유지, Data Mesh(팀별로 데이터 시스템을 유지하되 생기는 데이터들이 모든 팀에 보여야함)

개인식별정보
개인을 식별할 수 있는 정보 (성명, 전화번호, 주소, 신용카드 정보 .. )
+준식별자 : 조합으로 개인 식별할 수 있는 경우 (나이, 직장, 사는 도시 ..)
= 이것들이 보호의 대상

개인정보 보호
개인정보를 '동의없이' 저장, 노출, 배포
여러 회사에서 약관을 통해 개인정보에 대한 보호를 지키려고 노력중이다.
내부자 / 외부위협
>> 개인정보는 불필요하게 저장되면 안된다(필요한 것만 저장) .. 필요한 사람이 필요할때 저장하고 모든 엑세스를 워딩과 기록을 해서 나중에 감사, 오딧(Audit)할 수 있게 해야한다.

개인정보 보호법
GDPR : 유럽연합의 개인정보보호 법령(데이터 카탈로그, 거버넌스).. 필수
+위반 페널티
lv 1,2 : 행정처분 (과징금)
+큰회사들은 독자적인 시스템을 만들어 대응
*CPRA=CCPA : 캘리포니아 거주민으로 국한된 GDPR과 유사한 개인정보보호법 .. 개인+가구(household) 까지 정보보호
*HIPPA : 개인의 의료정보 보호 .. CCPA나 GDPR과 흡사
EHR/EMR, ePHI .. 최신 데이터와 부족하고 구조화된 데이터가 아니라 의사마다 다름 .. 수기기록을 이미지화 해서 받는 경우도 있음
