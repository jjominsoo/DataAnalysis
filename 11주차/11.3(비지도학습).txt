1. 군집화(클러스터링)
유사특성의 서브그룹/클러스터 분할
구조,패턴 발견 + 의미부여(도메인 지식을 가진 사람)

군집화는 단지 수단일 뿐. ( 군집화를 통해서 무엇을 알아내는 것이 아닌 특정 문제때문에 군집화를 진행시켰다로 가야함 )
ex) 고객 세분화(클러스터링)를 한 이유(마케팅전략, 신규고객유치 .. )를 설명할 줄 알아아함
스케일링이 중요
1-1. K-means	: K개의 클러스터로 나눔
1-2. 계층적 군집화	: 모든 데이터를 하나의 클러스터로 두고 유사한거끼리 묶는다
1-3. DBSCAN	: 데이터 밀도를 바탕


2. 차원 축소
고차원 데이터를 저차원 데이터로 만듬 ( 어느정도 핵심적 특징을 갖고 )
데이터 시각화, 계산 효율성 증가, 노이즈 감소, 더 나은 데이터 해석, 중요한 정보 강조

2-1. PCA		: 가장 분포가 큰 feature를 기준으로 차원을 축소함 ( 정보를 최대한 유지 )
2-2. t-SNE	: feature들의 관계(구조적특성)를 유지하면서 축소 ( 시각화에 유리 )
2-3. 오토인코더	: 인코더를 통해 고차원 데이터를 저차원으로 압축하고 디코더를 통해 원본을 되돌린다.


3. 이상 탐지
이상치 탐지
특성에서 많이 벗어난 데이터 식별 ( 금융 보안 의료 )

3-1. Isolation Forest	: tree형식으로 모든 데이터를 격리시킴 ( depth가 적을수록 뭉쳐있지 않다는 뜻 )
3-2. One-Class SVM	: '정상'으로 label된 데이터들을 학습하고, 정상으로 분류되지 않은 것들 탐지
3-3. LOF			: 밀도 계산 후 낮은 곳들을 탐지
