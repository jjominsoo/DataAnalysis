1. 군집화(클러스터링)
유사특성의 서브그룹/클러스터 분할
구조,패턴 발견 + 의미부여(도메인 지식을 가진 사람)

군집화는 단지 수단일 뿐. ( 군집화를 통해서 무엇을 알아내는 것이 아닌 특정 문제때문에 군집화를 진행시켰다로 가야함 )
ex) 고객 세분화(클러스터링)를 한 이유(마케팅전략, 신규고객유치 .. )를 설명할 줄 알아아함
스케일링이 중요

2. 차원 축소
고차원 데이터를 저차원 데이터로 만듬 ( 어느정도 핵심적 특징을 갖고 )
데이터 시각화, 계산 효율성 증가, 노이즈 감소, 더 나은 데이터 해석, 중요한 정보 강조

3. 이상 탐지
이상치 탐지
특성에서 많이 벗어난 데이터 식별 ( 금융 보안 의료 )







1.1 K-means Clustering
전체를 K개의 덩어리로 나눔

로이드 알고리즘 / 엘칸 알고리즘
클러스터링을 위한 알고리즘
1. 초기화 2. 할당 3. 업데이트 4. 반복
1. 랜덤한 K개의 중심점을 찍음 ( 최대한 서로 멀게 )
2. 각 중심점부터 나머지 데이터들의 거리를 계산 (유클리드거리 or 코사인 유사도 or 맨해튼 거리) + 거리마다 군집화
3. 새로 생긴 군집의 중심점 업데이트
4. 2~3 반복 > 중단점은 여러가지가 있음 ( 지정횟수, 변하지않는 중심점, 변하지않는 데이터 )

엘보우방법
적절한 K를 찾기위한 방법
그냥 K수를 늘려가면서 성능비교 > SSE (중심점과 데이터들 사이의 거리의 제곱합)
K가 많아질수록 SSE는 줄어드는데 그 타협점을 찾는 것이 중요

실루엣계수
응집도(같은 클러스터 내 데이터의 평균거리) 분리도(가장 가까운 다른 클러스터 중심과 데이터의 평균거리) >> -1 최악 1 최고 .. 0.5만 되도 좋은 분석
