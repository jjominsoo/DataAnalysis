Survivorship Bias : 대놓고 문제인 부분보다 아니라고 생각하는 부분에 문제가 있을 수도 있다. (비행기 피격점, 고객의 불만전화)
Confirmation Bias : 대놓고 문제인 부분을 집중적으로 파는 것 (집중수사)
>> 가설에서 생각하는 문제보다 더 큰 문제가 있는지 검토가 필요

>> 반복적인 질문은 대시보드에서 self service를 통해 직접 알 수 있도록 하자(Scalability)
+스킬셋 : SQL, python, ELT(T) .. 스키마정의=모델링, dbt툴, 통계적지식, A/B테스트, 지표정의, 대시보드, 비즈니스도메인 깊은 지식

Micro Service
웹개발에서 사용하는 기술. 하나의 서비스가 아닌 목적별로 다수의 서비스를 구현하고 서로 연동하도록 함. API로 나눠 적합한 언어나 프레임워크로 구성한다. 
+Service Registry를 작성하여 각 micro service정보 저장(data mesh - 카탈로그)

데이터 팀 일주일 업무
+Agile model : 개발 기간을 짧은 사이클(Sprint = 1~3주)로 나눠, 구체적으로 작은 기능을 바로 쓸 수 있을 정도로 구현해놓는다. + JIRA툴 사용

KPI <1일차 복습>
조직에서 달성하고자 하는 중요한 목표, 정량적 숫자, 명확한 정의 .. 지표사전 필요. 적을수록 좋음. 개인,팀별로 다름
기준) 가치를 나타내야함(총매출), 지속적인 가치 창출(달마다 매출 / 고객패턴 확인(재방문율)), 후행지표(과거의 데이터를 기준으로 분석), 피드백성
+의사결정장애 : 결정을 내리지 않고 새로운 대시보드만 만들어냄 .. 대시보드도 KPI처럼 적을수록 좋음
